# 视频会员系统（VOD-VIP）开发需求文档

## 1. 项目定义

### 1.1 项目目标
开发一个视频点播平台，核心逻辑为“内容付费”：用户需充值 VIP 才能解锁完整视频资源。

### 1.2 用户角色
- **访客**：仅可浏览视频列表，不可播放完整视频。
- **普通用户**：可播放免费内容（`is_vip = 0`）。
- **VIP 会员**：可播放全部内容（含 `is_vip = 1`）。

---

## 2. 技术栈与系统架构（推荐）

### 2.1 后端
- PHP `8.2+`
- 框架：Laravel `10` 或 ThinkPHP `6`

### 2.2 前端
- Vue `3`
- Tailwind CSS（响应式布局）

### 2.3 存储与缓存
- 数据库：MySQL `8.0`
- 缓存：Redis（用户 Session、视频热度统计）

---

## 3. 数据库模型设计（Schema）

### 3.1 视频资源表：`videos`

| 字段名 | 类型 | 描述 |
| --- | --- | --- |
| id | int | 主键 ID |
| title | varchar | 视频标题 |
| cover_url | varchar | 封面图链接 |
| video_url | varchar | 完整视频链接（已有资源路径） |
| preview_url | varchar | 30 秒试看片段链接（可选） |
| is_vip | tinyint | 0：免费；1：VIP 专属 |
| category_id | int | 分类 ID |
| description | text | 视频简介 |

### 3.2 用户表：`users`

| 字段名 | 类型 | 描述 |
| --- | --- | --- |
| id | int | 主键 ID |
| nickname | varchar | 用户昵称 |
| email/phone | varchar | 登录账号 |
| password | varchar | 加密密码 |
| vip_level | int | 0：普通；1：会员 |
| vip_expired_at | datetime | 会员到期时间 |

---

## 4. 核心功能与业务流程

### 4.1 视频控制逻辑（后端）

#### 流程说明
1. 接收视频详情请求（包含视频 ID）。
2. 查询视频资源，读取 `is_vip` 状态。
3. 若 `is_vip = 0`，直接返回完整播放地址 `video_url`。
4. 若 `is_vip = 1`，校验当前登录用户 `vip_expired_at > now()`。
5. 校验失败：返回 `403 Forbidden` 或仅返回 `preview_url`。
6. 校验通过：返回 `video_url`。

#### 鉴权建议
- 使用鉴权中间件统一处理 VIP 资源访问校验。
- API 返回中明确标识 `can_play_full`，便于前端展示状态。

### 4.2 充值与会员更新

#### 流程说明
1. 用户选择 VIP 套餐（如：月度、年度）。
2. 调用支付接口（微信/支付宝/聚合支付）。
3. 支付成功后，由支付回调（Webhook）通知后端。
4. 后端校验签名并处理业务：
   - 更新 `vip_level = 1`
   - 累加会员到期时间（示例：`vip_expired_at = now()->addMonths(1)`）

#### 续费规则建议
- 若当前仍在会员期内：从原到期时间继续累加。
- 若当前已过期：从当前时间重新计算到期时间。

---

## 5. 前端实现建议（Vue 3 + Tailwind）

### 5.1 列表页布局
- 使用 Tailwind 响应式网格：`grid-cols-2 md:grid-cols-4`
- 在移动端显示 2 列，桌面端显示 4 列。

### 5.2 VIP 标签
- 在封面右上角添加 `VIP` 标识。
- 使用 `absolute` + 黄色背景提升识别度。

### 5.3 播放器拦截体验
- 播放器可选：`video.js` 或 `artplayer`。
- 非 VIP 用户播放 VIP 视频时：
  - 播放至第 30 秒自动暂停；
  - 弹出“扫码充值”遮罩层，引导购买。

> 注意：前端拦截仅用于用户体验，不能作为权限判断依据。

## 6. 给 AI 开发者的指令（Prompt 示例）

### 指令 A：生成后端 API
> 请使用 Laravel 10 为我编写一个视频详情 API。逻辑如下：根据视频 ID 查询视频信息，如果 `is_vip` 字段为 `1`，则必须检查当前登录用户的 `vip_expired_at` 是否大于当前时间。如果不满足条件，不返回 `video_url`，只返回 `preview_url`。

### 指令 B：生成数据库迁移脚本
> 请为我生成 MySQL 的 Migration 脚本，包含用户表（带 VIP 到期时间）和视频表（带 VIP 状态和视频路径）。

---

## 7. 安全与风控建议

### 7.1 URL 签名
- 视频地址不要直接暴露。
- 后端应生成带过期时间的临时签名 URL，防止盗链。

### 7.2 并发与设备限制
- 限制单个 VIP 账号可同时在线设备数量，防止账号共享。

### 7.3 防绕过机制
- 所有播放权限必须由后端二次校验。
- 前端校验仅用于展示和交互，不可作为最终准入条件。

---

## 8. 接口返回建议（可选）

建议视频详情接口统一返回以下字段，便于前端快速处理：

- `id`
- `title`
- `cover_url`
- `is_vip`
- `can_play_full`（布尔值）
- `play_url`（满足权限返回 `video_url`，否则返回 `preview_url`）
- `vip_required_message`（如“开通 VIP 观看完整版”）

---

## 9. 验收标准（MVP）

- 免费视频可被访客与普通用户正常播放。
- VIP 视频对非会员不可播放完整内容。
- 会员支付成功后可立即解锁 VIP 内容。
- 会员过期后自动失去 VIP 播放权限。
- 接口在越权访问时返回明确状态码或可控降级数据。
